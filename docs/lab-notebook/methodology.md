# Methodology & reproducibility

This document is the load-bearing reference for everything in the lab notebook.
Read it before the chapters: it describes the data, the splitting strategy,
the per-fold pipeline, and — most importantly — exactly **how we guarantee
no data leakage** between training and test subjects.

All scripts referenced below live under `scripts/` and `slurm/` in the repo
root. Every reported number was generated by a SLURM job whose script and
output log are checked in (logs are under `slurm/logs/`, gitignored).

## 1. Data

### 1.1 Source

- **178 EEG recording sessions** of weighted Symbolic Mutual Information
  (wSMI), theta band, computed by Pablo's group and re-distributed by Greta
  via Google Drive on 2026-05-26.
- **256 EGI electrodes** (HydroCel system, GSN-HydroCel-257 with one

```

    └── 256_electrodes/
```

The 15-fold chunked layout is a Drive zip-split artefact. The session
enumerator `src/preprocessing.EEGtoGraph.enumerate_matrix_sessions` walks
the tree recursively to find every `sub-X/ses-Y/*.npz` regardless of how

(verified once at ingest).

Each `.npz` stores a single key `'data'` of shape `(n_epochs, 256, 256)`,
float32. The diagonal is zero (self-wSMI not defined), values are in
roughly `[0, 0.3]` range.

## 2. Subject-level splits — no data leakage

### 2.1 The splitting unit is always the SUBJECT

Every cross-validation split in this notebook uses
`sklearn.model_selection.GroupKFold` with `groups = subject_id`. This
guarantees that **all epochs of a given subject land in exactly one fold**
— either all in train or all in test, never split.

This is essential for DOC EEG: within-subject epochs are highly correlated
(same brain, same recording session, adjacent time windows). Splitting
epochs randomly would leak subject identity into the test set and inflate
all scores. Splitting by subject answers the real question — *"can the
model classify a new patient it has never seen?"*.

### 2.2 The 144 subjects → 5 folds split

`GroupKFold(n_splits=5)` over 144 subjects gives test folds of 28–29
subjects each. Sum across folds = 144 = total subject count: every subject
appears in exactly one test fold across the 5 folds. **No subject is in
the test set twice; no subject is in both train and test of the same
fold.** We log `n_train_subjects` and `n_test_subjects` per fold in
`output/holdout*/summary.json` for verification.

LOOCV (`n_folds=144`) is the same idea taken to the limit: each fold has
exactly one test subject.

### 2.3 Per-fold pipeline — what is fit on what

For each `(train_row_idx, test_row_idx)` pair (rows index epochs, not
subjects, but the GroupKFold guarantees subject-disjointness):

| step | code | fit/transform | **sees** |
|---|---|---|---|
| 1 | `scaler = StandardScaler().fit(X_raw[train_row_idx])` | **fit** | train epochs only |
| 2 | `X_train_scaled = scaler.transform(X_raw[train_row_idx])` | transform | train (no fit) |
| 3 | `pca = PCA(50).fit(X_train_scaled)` | **fit** | train epochs only |
| 4 | `X_pca_full = pca.transform(scaler.transform(X_raw))` | transform | both, no fit |
| 5 | mask `last_100` per session in `df_train`, balance by diagnosis | — | train only |
| 6 | `gm = GaussianMixture(K).fit(X_pca_full[train_bal_idx])` | **fit** | train only (balanced last_N subset) |
| 7 | `c2dx = cluster_to_dx_majority(gm.predict(train_bal), train_dx)` | derived | train only |
| 8 | build per-subject fingerprints on `df_train_last`, fit LogReg | **fit** | train fingerprints only |
| **TEST** | `gm.predict(X_pca_full[test_last_idx])` | predict | uses train-fit GMM |
| **TEST** | LogReg.predict on test fingerprints | predict | uses train-fit classifier |

**Four `.fit(...)` calls per fold (scaler, PCA, GMM, LogReg), all on train
rows only.** The fact that `X_raw` and `X_pca_full` exist in memory for the
whole dataset is not leakage — it's just convenience for indexing. Test
rows go through *frozen* pre-processors learned from train.

If you'd like the more paranoid (slower) equivalent, this is identical:

```python
X_train_pca = pca.fit_transform(scaler.fit_transform(X_raw[train_row_idx]))
X_test_pca  = pca.transform(    scaler.transform(    X_raw[test_row_idx]))
```

— and we verified empirically it gives the same numerical result.

### 2.4 What we explicitly DO NOT do

- ❌ Fit StandardScaler or PCA on the union of train+test.
- ❌ Compute the cluster→diagnosis lookup table with test subjects' labels.
- ❌ Apply `balanced_by_diagnosis` downsampling to the test set (test set
  remains naturally distributed; only the training GMM is fit on a
  diagnosis-balanced subset to prevent UWS-dominance).
- ❌ Split a single subject across train and test (`GroupKFold` enforces).
- ❌ Use `LeaveOneOut` (epoch-level) — that would mix subjects.

### 2.5 Hyperparameters / random seeds — frozen for reproducibility

Every numerical result in the notebook was generated with
`random_state=42`, threaded everywhere:

```python
scaler = StandardScaler()                                # deterministic, no seed
pca = PCA(n_components=50, random_state=42, svd_solver='randomized')
gmm = GaussianMixture(K, covariance_type='full', random_state=42,
                      n_init=3, max_iter=200, reg_covar=1e-3)
logreg = LogisticRegression(max_iter=2000, class_weight='balanced',
                            solver='lbfgs', random_state=42)
```

Notes:
- `n_init=3` GMM initialisations; the best (highest log-likelihood) is
  kept. Different random_states would slightly shuffle which cluster is
  labelled `0` vs `1` etc., but the structure is stable.
- `reg_covar=1e-3` was settled on after a tiny outlier cluster appeared at
  `1e-4`. See [chapter 3](./chapter_03_optimisations.md).
- `class_weight='balanced'` for LogReg: inverse-frequency weighting so
  rare diagnoses (COMA, EMCS) don't get drowned by UWS.

## 3. Common preprocessing across the notebook

### 3.1 The wSMI flat-vector representation

For raw-wSMI clustering experiments (chapters 1–3), each epoch is
represented as the **upper-triangle of its 256×256 wSMI matrix**, i.e. a
32,640-dim float32 vector (256·255/2). The diagonal and lower triangle are
redundant because wSMI is symmetric and zero on the diagonal.

This 32,640-dim vector then goes through:
1. `StandardScaler` — zero mean / unit variance per feature
2. `PCA(50)` — reduces to a 50-dim representation
3. clustering (KMeans / GMM / Spectral / Louvain)

PCA(50) captures ~12% of the variance on the full dataset, ~32% in
PCA(2) on the `last_100` subset. The signal is high-dimensional and
non-linear; this is why a learned representation (GAE, future work)
should help.

### 3.2 Cramér's V

We report Cramér's V as the effect-size metric for cluster × diagnosis
contingency tables. Definition:

  V = sqrt( chi² / (n · (min(K, n_dx) − 1)) )

It normalises χ² by sample size and degrees of freedom, so it's
comparable across tables of different shape. Rule-of-thumb:

- V < 0.10 — very weak association
- 0.10 ≤ V < 0.20 — weak
- 0.20 ≤ V < 0.35 — moderate
- V ≥ 0.35 — strong

With n ≈ 17,500 our p-values are always vanishingly small; **read the V,
not the p**.

### 3.3 Three prediction strategies (held-out evaluation)

After GMM fitting in a fold, each test subject's diagnosis is predicted in
three independent ways:

**Method A — modal-cluster → diagnosis lookup**
- During training, each cluster gets mapped to its dominant diagnosis.
- A test subject's "predicted diagnosis" = majority diagnosis of their
  modal cluster (the cluster most of their epochs fall into).
- Brittle if K < #diagnoses: some diagnoses may have no cluster mapping
  to them, or two clusters may both map to the same diagnosis.

**Method B — hard fingerprint → logistic regression**
- Each subject's "fingerprint" = a K-dim vector of cluster occupancy
  fractions (e.g. `[0.10, 0.05, 0.05, 0.80]`).
- A multi-class LogisticRegression learns
  `fingerprint → diagnosis` on training subjects, then predicts the
  diagnosis of held-out subjects from their fingerprint.

**Method C — soft posterior fingerprint → logistic regression**
- Same shape as B but uses the GMM's posterior probabilities
  `P(cluster | epoch)` averaged across the subject's epochs.
- Carries strictly more information than B when posteriors are diffuse.
  Empirically, GMM posteriors are peaky on our data, so B ≈ C.

**Coarsening 6→3 classes**
- For 3-class scoring we collapse via:
  - `control → control`
  - `UWS, COMA → low_doc`
  - `MCS-, MCS+, EMCS → high_doc`
- Both the LogReg-based methods (B, C) are trained directly on the
  3-class labels for the 3-class score (so the classifier learns the
  coarse boundaries).
- Modal-cluster (A) gets the coarse via a post-hoc collapse.

Baselines: random would give 1/6 ≈ 0.167 (6-class) or 1/3 ≈ 0.333
(3-class). We report **balanced accuracy** (`sklearn.metrics.balanced_accuracy_score`)
which is the mean recall per class, robust to imbalance.

## 5. Known caveats

1. **PCA(50) is fit on `X_train_raw` per fold but uses the same 32,640-D
   input across folds**. Numerical differences between folds reflect what
   the GMM finds in different train sets, not different feature
   pipelines.

2. **The `last_100_per_session` partition** is itself a hyperparameter
   (see [chapter 3](./chapter_03_optimisations.md) for the sweep). Other
   partitions give different V; `last_100` was the best in our sweep.

3. **The `cluster_to_dx_majority` lookup table is fold-dependent** —
   cluster IDs are arbitrary across runs (random_state-dependent). The
   *structure* (e.g. "one cluster is control-rich, another is
   UWS-rich") is stable across folds.

4. **Cramér's V can mislead under heavy class imbalance** — see
   chapter 1 for the discrepancy between in-sample V on balanced data
   (V=0.252) vs natural-imbalance prediction V (V=0.213).

5. **The wSMI matrices are theta-band only**. Other bands may carry
   complementary signal; not investigated here.

6. **No GAE / GNN in this notebook**. Chapters 1–3 are the raw-wSMI
   baseline. The learned-representation experiments are future work.
